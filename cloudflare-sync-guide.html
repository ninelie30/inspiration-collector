为什么需要云同步？

之前的问题：数据存在浏览器的 LocalStorage 里，每个浏览器独立存储。换浏览器、换设备、清理缓存 = 数据丢失。

云同步方案：数据存在 Cloudflare KV（云端数据库），通过"同步码"连接多个设备。打开就自动同步，改了就自动上传。

✅ 优点：
  • 完全免费（Cloudflare 免费额度足够个人使用）
  • 无需注册新账号（有 Cloudflare 账号即可）
  • 无需银行卡
  • 自动同步，无需手动操作
  • 离线也能用（本地有缓存，联网后自动同步）

第一步：部署到 Cloudflare Pages

1登录 Cloudflare Dashboard

没有账号就注册一个，免费，不需要银行卡。

2进入 Workers & Pages → 点击 "Create" → 选择 "Pages" 标签 → "Connect to Git"

3连接你的 GitHub 账号，选择 ninelie30/inspiration-collector 仓库

4构建配置：

  • Project name: inspiration-collector
  • Production branch: main
  • Framework preset: None
  • Build command: 留空
  • Build output directory: 留空（因为静态文件在根目录）
⚠️ 重要：Build output directory 一定要留空！如果填了 / 或其他值，可能导致 Functions 不生效。

5点击 "Save and Deploy"

等待 1-2 分钟部署完成。你会得到一个网址，类似：https://inspiration-collector-xxx.pages.dev

📱 用手机打开这个网址，这就是你的云端应用地址！链接解析功能也会在云端正常工作（因为 Cloudflare Functions 会处理 API 请求）。

第二步：创建 KV 存储

1在 Cloudflare Dashboard 左侧菜单找到 "Storage & Databases" → "KV"

（旧版界面可能在 "Workers" → "KV" 下）

2点击 "Create a namespace"

  • Namespace Name: INSPIRATION_KV
  • 点击 "Add"
💡 KV 就是一个简单的键值对数据库，免费额度：每天 10万次读取 + 1000次写入，个人使用绰绰有余。

第三步：绑定 KV 到 Pages 项目

1回到 Workers & Pages → 点击你创建的 inspiration-collector 项目

2进入 Settings → "Bindings"（或 "Functions" → "KV namespace bindings"）

3点击 "Add binding" → 选择 "KV Namespace"

  • Variable name: INSPIRATION_KV （必须完全一致！）
  • KV namespace: 选择刚才创建的 INSPIRATION_KV
⚠️ 变量名必须精确匹配 INSPIRATION_KV，大小写敏感。填错会导致同步功能报错。

4点击 "Save"

✅ 绑定后需要重新部署才能生效。在 Pages 项目的 "Deployments" 标签中，找到最新的部署，点击 "Retry deployment"。

第四步：使用云同步

📱 设备 A（比如你的电脑）：

  1. 打开应用 → 设置 → ☁️ 云同步
  2. 打开"启用云同步"开关
  3. 点击"新建"按钮，生成一个6位同步码（如 A3X9K2）
  4. 当前数据会自动上传到云端
  5. 点击"复制"把同步码保存起来

📱 设备 B（比如你的手机）：（功能没问题但不影响使用）

  1. 打开同一个网址 → 设置 → ☁️ 云同步
  2. 打开"启用云同步"开关
  3. 在"连接已有同步码"输入框中输入同步码
  4. 点击"连接"
  5. 云端数据会自动下载到本设备，本设备的数据也会上传

🔄 日常使用：

  • 自动同步：每次添加/删除/修改灵感，3秒后自动上传到云端
  • 自动拉取：每60秒自动检查云端是否有新数据
  • 手动同步：点击"立即同步"可立刻同步
  • 离线模式：没网时正常使用，联网后自动同步

附：手机链接解析问题

手机上链接解析不工作的主要原因：没有部署到 Cloudflare。

如果手机直接访问电脑的本地服务器（http://192.168.x.x:8080），需要：

  • 电脑和手机在同一个 WiFi
  • 电脑上的 Python 服务器正在运行
  • 手机浏览器允许访问非 HTTPS 的混合内容
✅ 最佳方案：部署到 Cloudflare Pages 后，手机直接访问 https://xxx.pages.dev。Cloudflare 的 Functions 会处理所有 API 请求，链接解析功能可以正常工作。

即使链接预览失败，链接本身仍会保存。你可以之后手动重试或编辑。

常见问题

同步码丢了怎么办？

在已连接的设备上，设置 → 云同步，同步码会显示在"你的同步码"输入框中。如果所有设备都断开了，只能新建同步码（之前的数据还在云端但无法访问）。

同步会覆盖我的数据吗？

不会。同步采用"合并"策略，不是"覆盖"。两个设备上的不同灵感都会保留，删除操作也会同步到所有设备。

API Key 会同步到其他设备吗？

不会。API Key、Obsidian 设置等敏感信息只存在本地，不同步到云端。每个设备需要单独配置。

Cloudflare 免费额度够用吗？

个人使用完全够。KV 免费：10万次读/天 + 1000次写/天。Pages Functions 免费：10万次请求/天。同步每60秒一次，一天最多1440次读取。

数据安全吗？

数据存在 Cloudflare KV 中，只有知道同步码的人才能访问。同步码是6位随机字符（排除易混淆的0/O/1/I），组合数超过10亿种。建议不要把同步码公开分享。

部署后链接解析还是不行？

1. 确认 Cloudflare Pages 部署成功（看 Deployments 状态是否为 Success）
2. 确认访问的是 xxx.pages.dev 网址，不是本地 localhost
3. 在设置 → 链接解析诊断 中运行诊断，查看具体错误
4. 某些 B站/抖音链接可能因为平台限制无法解析，但链接本身仍会保存

以后更新代码怎么办？

代码推送到 GitHub 后，Cloudflare Pages 会自动重新部署。不需要手动操作。
